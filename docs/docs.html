Overview

The Subvocal SDK is an open-source, hardware-agnostic middleware platform designed to connect surface electromyography (sEMG) biosensors directly to LLM-driven AI agents.

Traditional silent speech interfaces lock developers into proprietary neckbands or restricted, pre-trained whole-word vocabularies. Subvocal SDK overcomes these limits by providing the software infrastructure (digital signal preprocessing, deep learning training skeletons, articulatory phonetic shorthand simulators, and context-aware decoders) to enable open-vocabulary control with high accuracy and low execution latency.

Installation

The Subvocal SDK runs fully offline and does not require heavy runtime dependencies. You only need a standard python environment with packaging for data modeling: 

pip install pydantic scikit-learn joblib torch

Note: PyTorch models are configured by default to run on CPU threads. This avoids MPS device-pooling overhead on Apple Silicon, keeping gesture inference latencies under 1 millisecond.

Verification

To confirm that the ML classifiers, DSP pipelines, and decoders compile and run correctly, run the integration test suites: 

# Test 1: ML Classifiers (RF, CNN, GRU)
python3 sdk/emg_core/test_ml.py

# Test 2: Shorthand Phonetic Alignment
python3 sdk/shorthand/test_shorthand.py

# Test 3: Context-Aware Intent Resolution
python3 sdk/context/test_context.py

Benchmarks

The SDK comes pre-packaged with an evaluation harness that simulates muscle biopotential noise and executes intent-reconstruction tests across 50 realistic shorthand command scenarios: 

python3 sdk/shorthand/eval_runner.py

Running this script will output overall classification heuristics, word error rates, and average classification latencies.

Repository Structure

The codebase is organized in a clean, modular structure:

  • sdk/emg_core/: Signal preprocessing (notch and bandpass filtering) and classifier architectures (Random Forest, GRU, 1D CNN).
  • sdk/shorthand/: Compressed phonetic consonant simulators, dictionary vocabs, and the Asymmetric Levenshtein decoder.
  • sdk/context/: Pydantic context schemas and search priorities (calendar events, focused browser URL, contacts).
  • sdk/tts/: Local zero-dependency Text-to-Speech audio confirmations.

Digital Signal Preprocessing (DSP)

Physiological raw signals ingest at 250 Hz. Signal conditioning routines are located in sdk/emg_core/dsp/filters.py and apply:

  • An AlterEgo-inspired 1.3–50.0 Hz bandpass filter designed to capture slow, low-velocity articulatory gestures.
  • A **50 Hz notch filter** to remove local AC line noise.
  • Time-domain feature extraction yielding **TD10 segment features** (840-dimensional representations per segment).

ML Classifier Models

Custom pipeline training skeletons are located in sdk/emg_core/ml/train.py. The SDK supports:

  • Random Forest: Standard lightweight heuristic baseline.
  • GRU (Gated Recurrent Unit): Sequence model trained on temporal sEMG feature frames.
  • 1D CNN: Temporal convolutional model mapping multi-channel muscle traces.

Inference is managed by a unified InferenceEngine in sdk/emg_core/ml/infer.py utilizing confidence thresholds and adaptive cooldown gating to prevent duplicate gesture triggers.

Articulatory Shorthand Decoder

Rather than classifying whole words (which degrades rapidly beyond small vocabularies), users speak compressed phonetic consonant shorthand (e.g. g gl for Google).

The decoder in sdk/shorthand/decoder.py aligns shorthand strings using a custom dynamic programming cost matrix (Asymmetric Levenshtein) configured with biological muscle confusion groups (Labial, Alveolar, Velar, Rhotic). Vowels are ignored, and consonant omissions are weighted by physical gesture overlap.

Zero-Dependency TTS Engine

Located in sdk/tts/engine.py, the speech feedback system operates offline with zero external service requirements. It defaults to macOS system commands (say and afplay) to provide local audio confirmations.
